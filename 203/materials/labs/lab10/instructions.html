Lab 10, CSC 203

This lab explores streams through the implementation of a program to read in and transform a set of points, where each point is represented by an x, y and z value. These points can be drawn to form an image. The initial set of points includes a lot of "noise" - random points that obscure the image. Through implementing various computational operations on these points they are transformed into a more interesting image.

Streams are a useful tool to process collections. In particular, they allow operations to be carried out in parallel, e.g. on a multi-core computer. We don't need to do that here because our data set is small, but it's a valuable technique to learn. Streams can be used even in situations where parallel processing is not desirable.

A Java stream lets you operate on a collection of data using a sequence of commands and/or operations. You can read more about them here: https://docs.oracle.com/javase/8/docs/api/java/util/stream/package-summary.html .

Objectives

  • To gain familiarity with the streams API

  • To use stream operations including filter, map and collect.

Setup

The given code contains a program to read in the file positions.txt and create a PNG file called transformed.png. It then attempts to create a GUI that displays the image.

This data that is read represents a large number of points in space with an x, y and z value. You are to implement a method called transformPoints().

Stream API

After you have confirmed that you can read and process the points, work through the following operations using streams

  • Remove all points that have a z value > 2.0.

  • Scale down all the points by 0.5

  • Translate all the points by {-150, -37}

Be sure that you use collect, filter, map to complete these tasks. Collectors.toList() might come in handy.
