Lab 4, CSC/CPE 203 - Interfaces

Introduction

In this lab you will create a virtual drawing workspace on which you can place various geometric shapes. You will not actually be drawing the shapes in this program; you will simply be representing geometric shapes as objects placing them in a "workspace" data structure that represents a location in a 2D space. The workspace will make use of a java.util.List from the Java Standard Library to store shapes. It will have methods that allow you to add, remove, and inspect the shapes it contains. All of the shapes share a common set of methods defined by a Java interface. The implementation of these common methods varies, as appropriate, by shape. In addition, each shape has one or more methods unique to it. The following shapes will be supported: circle, rectangle, triangle, and convex polygon.

Objectives

  • To develop and demonstrate basic object-oriented development skills. Much of the structure of the solution is given - use good judgment when "filling in the blanks".

  • To become familiar with Java interfaces and the concept of polymorphism.

  • To practice using the instanceof operator.

  • More practice using existing classes and interfaces from the Java Standard Library, such as: List, ArrayList, LinkedList, and Color.

Given Files

In this lab, the only provided files are the Main class, a very simple Point class and a starter set of test cases. In the test cases file, the actual tests are commented out. That is to say, the code of the tests are made into a big multi-line comment, starting with /* and ending with */. You'll want to progressively remove test methods from within the comments, as you write more of the lab.

The given file is a starter test file for your code. Be sure to add to this file to make sure your code is working properly. The autograder will use a much more extensive test file.

Ground Rules

Many of the classes and interfaces you will be writing in this lab have counterparts in the Java Standard Library. You may not use the Java Standard Library versions in your implementation unless explicitly instructed to do so (see below). The Java Standard Library classes and interfaces you may use are identified by a complete package specification. For example, you will be using the java.awt.Color, java.util.List and java.util.ArrayList types from the Java Standard Library. You may also use the java.lang.Math class from the Java Standard Library. Any classes or interfaces in the specifications below (other than Color, List and ArrayList) must be entirely written by you. If you have any questions regarding this restriction please be sure to ask your instructor early in the development process.

As stated above, your classes and interfaces will use many classes that already exist in the Java Standard Library. For example, to use the Color classes, you will need to import java.awt.Color.

It might be possible to re-use some of your code from Lab 3, but it will need to be reused but will need to be re-written to match this specification.

Task

You must define the Shape interface and implement the Circle, Rectangle, ConvexPolygon, Triangle, and Workspace classes as described below. The instance variables for each class are not explicity specified. Remember, an instance variable makes every object of a class require more memory. All instance variables should be needed to represent the state of an object, or it should provide a significant computational efficiency or reduction in code complexity. On the other hand, local variables declared in the method in which they are used are inexpensive, and should be used freely. Be ready to justify your choices! Make sure all your instance variables are private, so that you maintain encapsulation.

You may add helper methods other than the methods specified below, but the additional methods must be private.

  1. Define a Java interface Shape with the following methods. (Note that the input parameters for each method, if any, are intentionally not shown. You must figure out what is needed):
    • Color getColor() - Returns the java.awt.Color of the Shape.
    • void setColor() - Sets the java.awt.Color of the Shape.
    • double getArea() - Returns the area of the Shape
    • double getPerimeter() - Returns the perimeter of the Shape
    • void translate() - Translates the entire shape by moving it by a given delta-x and delta-y, given as double parameters in that order. In a right-handed coordinate system, a positive delta-x moves to the right, and a positive delta-y moves up.

  2. Define a Java class Circle that implements the Shape interface. Circles have a radius, a center, and a Color. In addition to implementing the required methods specified by the Shape interface you must also implement the following methods. (Again, note that the input parameters for each method, if any, are intentionally not shown):

    • Circle() - A constructor with parameters to initialize the radius, center and color, in that order. Do not implement a zero-argument constructor.
    • double getRadius() - Returns the radius of the Circle.
    • void setRadius() - Sets the radius of the Circle
    • Point getCenter() - Returns the center of the Circle

    Note: Use the constant Math.PI when performing any calculations involving PI.

  3. Define a Java class Rectangle that implements the Shape interface. Rectangles have a width, a height, a Point indicating the top left corner, and a java.awt.Color. In addition to implementing the required methods specified by the Shape interface you must also implement the following methods. (Again, note that the input parameters for each method, if any, are intentionally not shown):

    • Rectangle() - A constructor with parameters for the width, the height, a Point representing the top-left corner, and a java.awt.Color, in that order. Do not implement a zero-argument constructor.
    • double getWidth() - Returns the width of the Rectangle.
    • void setWidth() - Sets the width of the Rectangle
    • double getHeight() - Returns the height of the Rectangle.
    • void setHeight() - Sets the height of the Rectangle
    • Point getTopLeft() - Returns the Point representing the top left corner of the Rectangle

  4. Define a Java class Triangle that implements the Shape interface. Triangles have three Points indicating its vertices and a Color. To simplify your solution, the vertices will always be given in counter-clockwise order. However, there are calculations for the area of a Triangle that will work with the vertices in any order. In addition to implementing the required methods specified by the Shape interface you must also implement the following methods. (Again, note that the input parameters for each method, if any, are intentionally not shown):

    • Triangle() - A constructor with parameters giving each vertex as a Point, and a java.awt.Color, in that order. The first Point passed to the constructor will be vertex A, the second Point vertex B, and so on.
    • Point getVertexA() - Returns the Point representing vertex A of the Triangle.
    • Point getVertexB() - Returns the Point representing vertex B of the Triangle.
    • Point getVertexC() - Returns the Point representing vertex C of the Triangle.

  5. Define a Java class ConvexPolygon that implements the Shape interface. A ConvexPolygon must a java Point array (not an ArrayList) indicating its vertices and a Color. Assume the Points in the Point array will always be given in counter-clockwise order. In addition to implementing the required methods specified by the Shape interface you must also implement the following methods. (Again, note that the input parameters for each method, if any, are intentionally not shown):

    • ConvexPolygon() - A constructor taking a Java Point[] and a java.awt.Color, in that order. Do not implement a zero-argument constructor.
    • Point getVertex() - Takes an index and returns the specified vertex of the ConvexPolygon. It should throw an exception if the index is not legal
    • int getNumVertices() - gives the number of vertices

  6. Define a Java class WorkSpace to hold Shape objects. The WorkSpace class shold have one private instance variable of type List<Shape> to hold Shape objects. You may choose which kind of List you use. The WorkSpace class should have the following public methods. (Note that this time the parameters are specified!):

    • WorkSpace() - A zero-argument constructor to initialize its instance variable to an empty List. The constructor should not take any parameters. (If you do nothing in this constructor, you may omit it from your code altogether.)
    • void add(Shape shape) - Adds an object which implements the Shape interface to the end of the List in the WorkSpace.
    • Shape get(int index) - Returns the specified Shape from the WorkSpace.
    • int size() - Returns the number of Shapes contained by the WorkSpace.
    • List<Circle> getCircles() - Returns a List of all the Circle objects contained by the WorkSpace.
    • List<Rectangle> getRectangles() - Returns a List of all the Rectangle objects contained by the WorkSpace.
    • List<Triangle> getTriangles() - Returns a List of all the Triangle objects contained by the WorkSpace.
    • List<ConvexPolygon> getConvexPolygons() - Returns a List of all the ConvexPolygon objects contained by the WorkSpace.
    • List<Shape> getShapesByColor(Color color) - Returns a List of all the Shape objects contained by the WorkSpace that match the specified Color.
    • double getAreaOfAllShapes() - Returns the sum of the areas of all the Shapes contained by the WorkSpace.
    • double getPerimeterOfAllShapes() - Returns the sum of the perimeters of all the Shapes contained by the WorkSpace.

    Use the instanceof operator in the appropriate places of your solution.

Suggestions

The following suggestions might be helpful:

  • Write the Shape interface first.

  • After creating the Shape interface, write one class at a time that implements the interface. Test and debug until you are satisfied with its design, implementation, and functionality. Then move on to the next class.

  • Implement the WorkSpace class last, and develop it incrementally, testing as you go. Notice that many of the methods are similar and differ only by the Shape type that they work with.

  • The given TestCases tests the signatures of each of your classes. As you implement classes, remove the comment characters to run the tests.

  • Mathwords is a good reference for the area of a convex polygon.
